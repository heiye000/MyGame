## 2. 当前工程现状（换机后先核对）

### 2.1 显示与像素（A3 基本已配）

- 逻辑分辨率 `640×360`
- `stretch/mode=viewport`，`aspect=expand`，`scale_mode=integer`
- 默认纹理过滤 Nearest（`default_texture_filter=0`）

### 2.2 玩家

- 场景：`res://actors/player/player.tscn`
- 脚本：`res://actors/player/player.gd`（`class_name Player extends CharacterBody2D`）
- 结构（已扁平化）：

```text
Player (CharacterBody2D)          # 组 Player；collision_layer=2(ActorBody)；motion floating
├── Sprite2D_Shadow / Sprite2D / CollisionShape2D
├── YSortable2D / Area2D / CameraTarget
├── InputBuffer（含调试叠层）
├── AnimationPlayer / AnimationTree（PlayerAnimationTree）
└── LimboHSM / NormalBattle
```

- 移动/攻击/翻滚：`normal_battle.gd` 直接操作 `player.velocity` / `move_and_slide()`
- 兼容：`player.character` 只读 getter 返回 `self`（旧代码可读）
- 走速仍约 `50`（规范建议日后调到约 `74`，**未做**）
- 方向动画仍主要是 BlendSpace / 四向攻击优先，**尚未**落到 `Direction8` 枚举模块

### 2.3 相机

- 脚本：`res://core/components/camera/world_camera.gd`（`class_name WorldCamera`）
- 场景：`res://core/components/camera/world_camera.tscn`（必须挂上述脚本）
- `test_world.tscn`：`WorldCamera` 为世界根子节点，`target = NodePath("../Player")`
- 玩家场景内 **无** `Camera2D`（有 `CameraTarget` 备用）
- **前视**（`set_move_look` 等）尚未接到玩家移动输入

### 2.4 世界 / 入口

- 启动链：`root` / `main` → `scenes/test/test_world.tscn`（灰盒）
- `scenes/world/World.tscn`：大地图灰盒 + YSort + 玩家实例，**尚无** `WorldCamera`（若改跑 World 需补接）
- **无** Chunk / 流式加载 / WorldRoot 正式结构

### 2.5 输入与调试（部分已有）

- GUIDE + `InputMappingScheme` + `InputBuffer` / 战斗缓冲
- `DebugService` + YSort / 输入缓冲叠层
- **无** 统一 EventBus、无区块调试、无命令控制台

### 2.6 敌人

- `BatEnemy` 仍是 `Node2D` 包 `CharacterBody2D`（玩家已改根，蝙蝠未改）
- 追击/视线已改为跟 `_player.global_position` / 碰撞体 `_player`

### 2.7 约定存放位置

- 碰撞层 / 渲染层 / ID / 脚底 / 八方向 / 相机 → **仅** `.cursor/rules/project-conventions.mdc`
- `docs/项目约定.md` **已删除**，勿再创建重复源

### 2.8 已知债务 / 注意

- 既有场景的 `collision_layer` / `mask` **数值大多未按新层表迁移**（只冻结了命名）
- `visibility_layer` 同理未迁移
- 编辑 `.tscn` 时避免 UTF-8 BOM（曾导致 `player.tscn` Parse Error: Expected '['）
- Windows 上注意 `Player.tscn` / `player.tscn` 大小写混用风险

## 3. 八方向正式命名（已冻结，实现时遵守）

```text
up, up_right, right, down_right, down, down_left, left, up_left
```

- 动画：`idle_up`、`walk_up_right` 等
- 实绘优先：`up, up_right, right, down_right, down`；镜像：`up_left, left, down_left`
- **禁止**再用 `n/ne/e/...` 作枚举名
