### 记录一些查询的信息
1.GUIDE Trigger方式
| Trigger            | 作用         | ARPG 推荐度 | 适合什么         |
| ------------------ | ---------- | -------- | ------------ |
| **Pressed**        | 按下瞬间触发一次   | ⭐⭐⭐⭐⭐    | 攻击、翻滚、技能、交互  |
| **Released**       | 松开时触发      | ⭐⭐       | 蓄力结束、弓箭发射    |
| **Hold**           | 按住达到一定时间   | ⭐⭐⭐⭐     | 蓄力攻击、长按奔跑    |
| **Tap**            | 快速点击       | ⭐⭐⭐      | Dash、双模式技能   |
| **Pulse**          | 按住期间周期触发   | ⭐⭐       | 连续射击、机关      |
| **Combo**          | 特定按键序列     | ⭐⭐       | 格斗游戏连招       |
| **Chorded Action** | 组合键        | ⭐⭐⭐      | Shift+A、LT+X |
| **Hair**           | 输入阈值（模拟摇杆） | ⭐        | 手柄           |
| **Stability**      | 输入稳定检测     | ⭐        | 手柄、触摸        |

2.可破坏的静态物品流水线需要考虑哪些？
物理碰撞分层    玩家受击区、敌人、拾取地区等不分层的话会误触
命中过滤        需要通过group / Hitbox 组件；加 _destroyed 防同帧双碎
碰撞默认值      缺省 collision=true + 圆心在贴图中心，对草过重、对树偏高 
可破坏默认      缺省 destructible=true，树/石头也会被当成可砍，改为显式开启
受击反馈        只有消失，没有音效/闪白/抖
掉落            ARPG 静物常掉材料   可选 loot_table / 掉落场景
销毁事件        任务、统计、存档需要知道「碎了」 signal destroyed
开放世界持久化   碎了重进 Chunk 又长出来 至少预留 persist_id / 与存档钩子
特效流水线衔接   死亡、其他的动画特效
