每个 tile = 16 × 16 px
所以一个 Chunk = 1024 × 1024 px

# chunk加载规则
玩家当前 Chunk：必须加载
周围 8 个 Chunk：保持加载
更远 Chunk：卸载

# 地图层级划分
GroundLayer        草地、土路、水面底层
TerrainLayer       山体、悬崖、河岸、桥
DecorationLayer    花草、石头、树影
CollisionLayer     阻挡瓦片，可以隐藏
OverheadLayer      树冠、屋顶、前景遮挡

如果你做俯视角 ARPG，最关键的是 Y Sort。
动态对象、树、建筑、NPC、玩家都要根据 Y 坐标决定遮挡关系。
Chunk
├── GroundLayer
├── TerrainLayer
├── WorldYSort
│   ├── Player
│   ├── NPCs
│   ├── Enemies
│   ├── Trees
│   └── Props
└── OverheadLayer
地面不参与 Y Sort，人物和可遮挡物参与 Y Sort，最前景单独放 OverheadLayer。


# 玩家角色划分
你应该从一开始就拆状态机 移动、攻击、翻滚、受击、死亡，分别放到不同 State 里。这样后期你加冲刺、蓄力攻击、骑马、游泳、爬梯子，不会把玩家脚本改炸

# 战斗系统
用组件
HitboxComponent      攻击判定
HurtboxComponent     受击判定
HealthComponent      血量
StatsComponent       攻击、防御、暴击、抗性
DamageInfo           伤害数据

武器 / 技能生成 Hitbox
        ↓
Hitbox 接触 Hurtbox
        ↓
生成 DamageInfo
        ↓
Hurtbox 把 DamageInfo 交给 HealthComponent
        ↓
HealthComponent 计算扣血、死亡、硬直


# 数据资源化
物品、怪物、技能、任务都不要写死在脚本里。
建议用 Godot Resource：
ItemData.gd
EnemyData.gd
SkillData.gd
QuestData.gd
DialogueData.gd
WeaponData.gd
ArmorData.gd

例如物品数据样例
extends Resource
class_name ItemData

@export var id: String
@export var name: String
@export var icon: Texture2D
@export var description: String
@export var max_stack: int = 99
@export var item_type: String
@export var value: int = 0

然后是每一个物品是一个.tres
data/items/item_potion_small.tres
data/items/item_iron_sword.tres
data/items/item_wood.tres
